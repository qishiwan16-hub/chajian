# Termux 下批量更新 SillyTavern 扩展脚本说明

这是一份给普通用户看的使用说明。你不需要懂脚本开发，只要跟着步骤做，就可以用这个脚本批量检查并更新 SillyTavern 的扩展。

---

## 一、这个脚本是做什么的

脚本文件是 `update_sillytavern_extensions_termux.sh`。

它的作用是：

1. 让你输入 **SillyTavern 根目录**
2. 让你输入 **用户目录名**
3. 自动检查两个扩展目录里的每一个子目录
4. 如果某个子目录本身是 Git 仓库，就尝试获取远程更新并拉取更新
5. 如果不是 Git 仓库，或者没有配置远程上游分支，就跳过
6. 最后输出本次检查结果统计

简单理解就是：

**它不是更新 SillyTavern 主程序，而是批量检查并更新扩展目录里的 Git 扩展。**

---

## 二、适合什么场景使用

这个脚本适合下面这些情况：

- 你在 Termux 里使用 SillyTavern
- 你安装过一些扩展
- 这些扩展是通过 Git 克隆下来的
- 你不想一个一个进入目录手动执行 `git pull`

如果你的扩展不是 Git 仓库，而是手动复制进去的文件夹，那么脚本会跳过它们。

---

## 三、使用前准备

在运行脚本之前，请先确认下面几件事。

### 1. 你已经安装了 Termux

这个脚本是给 **Termux** 用的，不是给 Windows 命令提示符、PowerShell 或 Linux 桌面终端直接用的。

### 2. 你已经安装了 Git

脚本运行时会用到 Git。

你可以先在 Termux 里执行：

```sh
pkg install git
```

如果已经装过，再执行一次也没关系。

### 3. 你知道自己的 SillyTavern 安装位置

脚本会让你输入 **SillyTavern 根目录**。

所谓根目录，简单理解就是：

**能看到 `public`、`data` 等文件夹的那个 SillyTavern 主目录。**

常见示例：

```text
/data/data/com.termux/files/home/SillyTavern
```

如果你的 SillyTavern 就安装在 Termux 默认家目录下，通常直接回车使用默认值就可以。

### 4. 你知道自己的用户目录名

脚本还会让你输入 **用户目录名**。

默认是：

```text
default-user
```

如果你没有改过，大多数情况下直接回车就行。

如果你改过用户目录名，就需要输入你实际使用的名字。

---

## 四、文件说明

当前要使用的脚本文件是：

```text
update_sillytavern_extensions_termux.sh
```

它是一个 Shell 脚本，专门为 Termux 环境准备，用来批量扫描并更新扩展目录中的 Git 仓库。

你不需要修改脚本内容，正常运行即可。

---

## 五、脚本会检查哪两个扩展目录

脚本会按顺序检查下面两个目录。

### 1. 第三方扩展目录

```text
<SillyTavern根目录>/public/scripts/extensions/third-party
```

例如：

```text
/data/data/com.termux/files/home/SillyTavern/public/scripts/extensions/third-party
```

这个目录通常用来存放公共的第三方扩展。

### 2. 用户扩展目录

```text
<SillyTavern根目录>/data/<用户名>/extensions
```

例如用户名是 `default-user` 时：

```text
/data/data/com.termux/files/home/SillyTavern/data/default-user/extensions
```

这个目录通常用来存放某个用户自己的扩展。

## 六、详细使用步骤

下面按最容易照着做的方式说明。

### 第 1 步：把脚本放到 Termux 能访问的位置

如果这个脚本已经在你的 Termux 工作目录里，可以直接用，不需要重复复制。

如果你是从别处拿到脚本，请把它放到你方便操作的位置，例如：

```text
/data/data/com.termux/files/home/
```

假设脚本放在家目录里，那么文件路径可能是：

```text
/data/data/com.termux/files/home/update_sillytavern_extensions_termux.sh
```

### 第 2 步：进入脚本所在目录

例如脚本在家目录，就执行：

```sh
cd ~
```

如果你把脚本放在别的位置，就进入对应目录。

### 第 3 步：给脚本执行权限

执行：

```sh
chmod +x ./update_sillytavern_extensions_termux.sh
```

这一步只需要做一次。以后脚本文件不变，一般不用重复授权。

### 第 4 步：运行脚本

执行：

```sh
./update_sillytavern_extensions_termux.sh
```

如果你不想先授权，也可以直接这样运行：

```sh
sh ./update_sillytavern_extensions_termux.sh
```

### 第 5 步：输入 SillyTavern 根目录

运行后会看到类似提示：

```text
请输入 SillyTavern 根目录（直接回车使用默认值：/data/data/com.termux/files/home/SillyTavern）：
```

这时你有两种做法：

- **直接回车**：使用默认目录
- **手动输入路径**：如果你的 SillyTavern 装在别的地方

例如：

```text
/data/data/com.termux/files/home/SillyTavern
```

也可以输入：

```text
~/SillyTavern
```

脚本会自动把 `~` 展开成你的家目录。

### 第 6 步：输入用户目录名

接着会看到类似提示：

```text
请输入用户目录名（直接回车使用默认值：default-user）：
```

这时你也有两种做法：

- **直接回车**：使用默认用户名 `default-user`
- **输入你自己的用户名目录**：如果你实际不是这个名字

例如：

```text
default-user
```

### 第 7 步：等待脚本检查和更新

输入完成后，脚本会先告诉你它准备检查哪两个目录，然后开始逐个检查子目录。

你会看到类似这样的过程输出：

```text
==== 扫描目录：第三方扩展目录 ====
/data/data/com.termux/files/home/SillyTavern/public/scripts/extensions/third-party

[检查] 某个扩展名
  -> [无更新] 本地已是最新
```

或者：

```text
[检查] 某个扩展名
  -> [已更新] 已拉取远程更新（远程领先 2 个提交）
```

或者：

```text
[检查] 某个扩展名
  -> [跳过] 不是 Git 仓库
```

运行结束后会输出一份汇总统计。

## 七、脚本运行逻辑说明

下面用简单的话解释它内部是怎么工作的。

### 1. 先检查 Git 是否存在

脚本启动后会先确认系统里有没有 `git`。

如果没有，它会直接提示错误，并告诉你安装命令：

```sh
pkg install git
```

### 2. 读取你输入的路径和用户名

- 如果你对根目录直接回车，就使用默认值 `~/SillyTavern`
- 如果你对用户名直接回车，就使用默认值 `default-user`
- 如果根目录最后带了 `/`，脚本会自动去掉多余的斜杠，避免路径拼接出错

### 3. 依次扫描两个总目录

脚本会先检查：

```text
public/scripts/extensions/third-party
```

再检查：

```text
data/<用户名>/extensions
```

### 4. 只遍历每个总目录下面的直接子目录

意思是：

脚本会看这两个扩展总目录里面的每一个文件夹，把每个文件夹当成一个候选扩展项目来处理。

它不会无限递归往更深层一路扫描。

### 5. 判断每个子目录是不是 Git 仓库

如果某个子目录不是 Git 仓库，脚本会：

- 记为跳过
- 输出：`[跳过] 不是 Git 仓库`

### 6. 如果是 Git 仓库，先执行 fetch

脚本会先执行类似 Git 获取远程信息的操作，相当于先看看远程有没有变化。

如果 `fetch` 失败，会：

- 记为失败
- 输出：`[失败] fetch 失败`

### 7. 检查仓库是否配置了上游分支

如果这个 Git 仓库没有设置远程跟踪分支，脚本无法判断该跟谁比较，就会：

- 记为跳过
- 输出：`[跳过] Git 仓库未配置上游分支`

### 8. 比较本地和远程差异

脚本会比较：

- 本地是否领先
- 远程是否领先

#### 情况 A：远程没有新提交

如果远程没有新内容：

- 记为无更新
- 输出可能是：
  - `[无更新] 本地已是最新`
  - 或 `[无更新] 远程没有新提交（本地领先 X 个提交）`

第二种情况表示：你的本地比远程还多了一些提交，但远程没有新的东西可拉。

#### 情况 B：远程有新提交

如果远程有更新，脚本会执行：

```text
git pull --ff-only
```

也就是只做 **快进式更新**。

简单理解：

- 能直接顺着远程更新，就自动更新
- 如果不能安全快进，就不会强行乱合并

如果更新成功，会：

- 记为已更新
- 输出：`[已更新] 已拉取远程更新`

如果 `pull` 失败，会：

- 记为失败
- 输出：`[失败] pull 失败`

## 八、输出结果怎么看

脚本最后会打印一段汇总统计，类似：

```text
==== 汇总统计 ====
已检查子目录：10
已更新仓库：3
无更新仓库：4
已跳过项目：2
失败项目数：1
```

下面分别说明这些数字代表什么。

### 1. 已检查子目录

表示脚本总共看了多少个扩展子目录。

只要是两个扩展总目录下的直接子目录，都会计入这个数字。

### 2. 已更新仓库

表示其中有多少个 Git 扩展成功拉取到了远程新内容。

### 3. 无更新仓库

表示这些 Git 扩展已经是最新状态，或者远程没有比本地更新的提交。

### 4. 已跳过项目

表示这些项目没有被实际更新检查到底，常见原因有：

- 不是 Git 仓库
- Git 仓库没有配置上游分支

### 5. 失败项目数

表示脚本尝试处理时出了问题，常见原因有：

- 网络异常
- Git fetch 失败
- Git pull 失败
- 仓库状态异常

如果这个数字不是 0，建议往上翻看对应扩展前面的报错内容。

## 九、常见问题与处理方法

### 问题 1：提示未检测到 Git

提示大致会是：

```text
错误：未检测到 git，请先在 Termux 中安装 git。
```

处理方法：

```sh
pkg install git
```

装完后重新运行脚本。

---

### 问题 2：输入路径后提示目录不存在或扫描不到内容

可能原因：

- 你输入的 SillyTavern 根目录不对
- 你输入到了扩展目录本身，而不是 SillyTavern 根目录
- 用户目录名不对

正确思路是：

你输入的应该是 **SillyTavern 主目录**，不是 `third-party` 目录，也不是 `extensions` 目录。

正确示例：

```text
/data/data/com.termux/files/home/SillyTavern
```

不是：

```text
/data/data/com.termux/files/home/SillyTavern/public/scripts/extensions/third-party
```

如果用户名不是 `default-user`，记得输入你自己的目录名。

---

### 问题 3：显示 `[跳过] 不是 Git 仓库`

说明这个扩展目录里没有 Git 仓库信息。

这通常表示：

- 这个扩展不是通过 `git clone` 获取的
- 或者 `.git` 信息已经丢失

处理方法：

- 如果你只是手动复制文件夹，这种跳过是正常的
- 如果你本来希望它能更新，需要确认这个扩展是否本来就是 Git 克隆下来的

---

### 问题 4：显示 `[跳过] Git 仓库未配置上游分支`

说明这个仓库虽然是 Git 仓库，但没有设置要跟踪的远程分支。

简单说：脚本不知道该和哪个远程分支比较，所以只能跳过。

处理方法：

- 如果你了解 Git，可以手动为该仓库设置上游分支
- 如果你不了解 Git，可以先记住这是仓库配置问题，不是脚本本身坏了

---

### 问题 5：显示 `[失败] fetch 失败`

常见原因：

- 网络不稳定
- 远程仓库地址失效
- 该扩展仓库需要特殊访问方式

处理方法：

1. 先确认网络正常
2. 过一会儿重新运行脚本
3. 如果总是失败，进入对应扩展目录手动执行 Git 命令查看详细原因

---

### 问题 6：显示 `[失败] pull 失败`

这通常说明仓库不能直接快进更新。

因为脚本使用的是：

```text
git pull --ff-only
```

这意味着它不会自动帮你做复杂合并。

处理方法：

- 如果你只是普通用户，可以先跳过这个扩展，避免误操作
- 如果你了解 Git，再手动进入该仓库处理分支分歧或冲突问题

---

### 问题 7：显示目录为空

如果某个扩展总目录下面没有任何子目录，脚本会提示该目录下没有可检查的子目录。

这一般不是错误，只是说明这个位置当前没有扩展可处理。

---

### 问题 8：默认值能不能直接用

大多数情况下：

- SillyTavern 装在 `~/SillyTavern`
- 用户目录名是 `default-user`

那么两次输入都可以直接回车。

如果你的安装方式和默认情况不同，再手动输入即可。

## 十、给普通用户的推荐操作方式

如果你不确定怎么填，建议按下面方式先试一次：

1. 在 Termux 里进入脚本目录
2. 执行：

```sh
chmod +x ./update_sillytavern_extensions_termux.sh
./update_sillytavern_extensions_termux.sh
```

3. 看到根目录提示时，先直接回车
4. 看到用户名提示时，再直接回车
5. 看脚本是否能正确列出你自己的扩展目录并开始检查

如果可以正常跑通，说明你的默认路径和默认用户名就是对的。

## 十一、可选自定义说明

如果你想少输入一些内容，也可以直接修改脚本里的默认值，但这只是可选操作，不改也能正常用。

脚本里有两个默认设置：

- `DEFAULT_ST_ROOT`：默认的 SillyTavern 根目录
- `DEFAULT_USER_NAME`：默认的用户目录名

如果你每次都用同一个路径、同一个用户名，可以把它们改成你自己的常用值。

不过对于普通用户来说：

**不改脚本也完全可以，运行时按提示输入就行。**

## 十二、最简使用示例

如果你的情况是：

- Termux 已安装 Git
- SillyTavern 在默认目录 `~/SillyTavern`
- 用户目录名是 `default-user`

那么最简单的使用流程就是：

```sh
cd ~
chmod +x ./update_sillytavern_extensions_termux.sh
./update_sillytavern_extensions_termux.sh
```

然后：

- 第一次提示直接回车
- 第二次提示直接回车
- 等待脚本输出检查结果

## 十三、一句话总结

这个脚本的作用就是：

**在 Termux 中，帮你批量检查并更新 SillyTavern 两个扩展目录里的 Git 扩展，最后告诉你哪些更新了、哪些没更新、哪些被跳过、哪些失败了。**
